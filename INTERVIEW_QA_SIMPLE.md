## 二、核心技术

### Q2: JWT是如何实现的？

**参考回答**：
```
【生成Token】
登录成功后，用JwtUtil生成Token，包含userId、username、role

【验证Token】
1. 前端请求携带：Authorization: Bearer {token}
2. JwtInterceptor拦截器验证Token有效性
3. 解析出用户信息，存入request

【安全措施】
- 密钥从配置文件读取，不硬编码
- Token有效期7天
- 密码用BCrypt加密，不放入Token
```

### Q3: 为什么选择JWT而不是Session？

**参考回答**：
```
JWT优势：
1. 无状态：服务端不用存储Session，适合分布式
2. 可扩展：Token包含用户信息，微服务间可传递
3. 跨域友好：不受同源策略限制

Session劣势：
1. 需要服务端存储，集群时要共享Session
2. 扩展性差

本项目用JWT + 拦截器实现认证，配合@RequireRole注解做权限控制。
```

### Q4: Redis缓存是怎么用的？

**参考回答**：
```
【缓存内容】
统计数据（分类统计、每日趋势），因为这些数据查询频繁但不常变化

【实现方式】
使用Spring Cache注解：
- @Cacheable：查询时缓存
- @CacheEvict：更新时清除缓存

【效果】
- 首次查询：~180ms（查数据库）
- 缓存命中：<3ms（查Redis）
- 提升约60倍
```

### Q5: 权限控制是如何实现的？

**参考回答**：
```
【实现方案】自定义注解 + AOP切面

1. 定义@RequireRole注解
2. RoleAspect切面拦截带注解的方法
3. 从request获取用户角色，检查是否匹配
4. 不匹配则抛出"权限不足"异常

【使用示例】
@RequireRole({"ADMIN"})
@GetMapping("/list")
public Result listLogs() { ... }

这样管理员接口只有ADMIN角色能访问。
```

**参考回答**：

```
【索引设计】
consumption_record表：
- idx_user_id：加速按用户查询

- 提升16倍

```

### Q7: 如何防止SQL注入？

**参考回答**：
```
【防护措施】
1. MyBatis使用#{ }预编译，不用${ }
2. MyBatis-Plus的LambdaQueryWrapper自动参数化
3. 所有用户输入都作为参数传递

【示例】
// ✅ 安全
wrapper.eq(ConsumptionRecord::getUserId, userId);

// ❌ 危险（本项目没这样做）
String sql = "SELECT * FROM record WHERE user_id = " + userId;
```

## 五、安全问题

### Q9: 密码是如何加密的？

**参考回答**：
```
【加密方案】BCryptPasswordEncoder

【为什么不用MD5？】
- MD5可逆（彩虹表攻击）
- 相同密码哈希值相同

【BCrypt优势】
- 自动加盐，每次生成的哈希值都不同
- 计算成本高，抗暴力破解

【实现】
注册时：passwordEncoder.encode(rawPassword)
登录时：passwordEncoder.matches(rawPassword, encodedPassword)
```

## 💡 面试技巧

2. **结合实际**：用项目的例子说明
3. **诚实坦诚**：不会就说不会，但可以展示思考过程
4. **主动引导**：往自己熟悉的方向引导
